Ways to Pull Repos into LLM Context

LLMs
Agents
Ways to let LLMs know about a GitHub repo or a local repo.
Author

Salman Naqvi

Published

Sunday, 25 January 2026

Good context is key when working with LLMs, no matter if you‚Äôre coding, writing, planning, problem solving, and what not. The LLM needs to know your context to you help you effectively.

This guide will explore different ways one can pull a GitHub repo or a local repo in particular into a format LLMs can use.

GitHubq

GitHubq is a web service from Answer.Ai that allows you to easily make GitHub repos available as context for LLMs. The basic way to use the service is to change github.com to githubq.com.

You can change what you get back with various parameters. For instance,

  • exts=py returns only files ending with .py
  • sigs_only=True returns only function signatures and their docstrings.

See the full documentation of Answer.AI‚Äôs toolslm library here.

repo2ctx

Githubq is great if you want to paste a link to a LLM that will then retrieve what‚Äôs on the link, or if you want to save one-off files here and there. What if you want to programmatically work with the context? That‚Äôs where toolslm, and more specifically the toolslm.xml module comes in.

from toolslm.xml import repo2ctx
print(repo2ctx('https://github.com/AnswerDotAI/shell_sage', exts='py')[:100])

contextkit

Next up is contextkit‚Äôs read module. However, this approach is much more hands on as the module‚Äôs functions do not return XML formatted contents. Though this does allow you to perform your own processing/handling. You will need to have a SSH key set up between your machine and GitHub as the functions in read create temporary clones of the GitHub repo.

contextkit.reads read_gh_repo returns a dictionary in the format of {'path': 'contents'}.

At an Overview

Method Input Best For
githubq Change URL to githubq.com Quick browser-based access
repo2ctx GitHub URL Programmatic access to remote repos
folder2ctx Local path Processing local folders
py2sigs Python file path Single file signatures
contextkit.read_gh_repo GitHub URL Raw dict of full repo contents
contextkit.read_gh_file File URL Single file contents
sym2pkgctx Class/function object Installed packages
sym2folderctx Class/function object Folder containing a symbol
read_url (SolveIt) GitHub URL Quick repo overview + README

Other toolslm.xml Goodies

Beyond the functions we‚Äôve covered, toolslm.xml has other interesting functions:

Additional Functions: - files2ctx ‚Äî Pass in a specific list of files rather than an entire folder - nb2xml ‚Äî Convert Jupyter notebooks to XML format - json_to_xml ‚Äî Convert JSON or Python dicts to XML - sym2file ‚Äî Get the source file contents for any class or function

Useful Parameters: - prefix=True ‚Äî Prepends Anthropic‚Äôs recommended preamble for document context - nums=True ‚Äî Adds line numbers to code (handy for referencing specific lines)

Command Line Support: Both folder2ctx and repo2ctx are available as CLI tools, so you can use them directly from your terminal without writing Python.

There‚Äôs plenty more to discover. Explore the full module at the docs by pulling it into a LLM with the techniques we‚Äôve learned here!
